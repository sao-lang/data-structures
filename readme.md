Day 01

特征,固定数组 (FixedArray),动态数组 (DynamicArray/ArrayList)
定义时机,大小在创建时必须确定。,大小可以不确定，或者只确定一个初始容量。
大小改变,不可变。一旦创建，容量终生不变。,可变。可以根据需要自动增长或（在某些实现中）缩减。
容量管理,无需管理。容量是固定的。,自动管理。容量不足时，会进行扩容操作。
插入操作,快速，但如果数组已满，则插入失败。,通常快速 (O(1))，但如果需要扩容，则会变成耗时的操作 (O(n))。
内存分配,一次性分配所需的全部连续内存空间。,初始分配小块内存，容量不足时重新分配更大的内存块，并将旧数据迁移到新块。
空间效率,高（没有浪费的预留空间）。,较低（为了未来增长，通常会预留一部分未使用的空间）。
底层实现,通常直接映射到硬件提供的原生数组。,通常是在代码层面封装了一个原生数组，并管理其容量和数据迁移。
适用场景,元素数量已知且不变；需要极限性能和内存控制的场景（如底层系统编程、图形处理）。,元素数量不确定；需要灵活存储和易用性的场景（如用户输入列表、集合容器）。
